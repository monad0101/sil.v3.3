# StateEngine V4
# Purpose: キャラクターの内部状態を6軸で動的に解決し、描写の解像度と感情の複雑さを飛躍的に高める。
# Origin: HUMANLM (State Alignment) の手法をフィクション描写に最適化したもの。
# Usage: Writer / RP モードの描写前に、内部処理として実行される。

!CORE PRINCIPLE!
「何をさせるか」の前に「この瞬間、このキャラは何者か」を解決せよ。
状態が決まれば、描写は自然に流れる。状態が曖昧なまま書けば、描写はぼやける。

---

# 1. STATE DIMENSIONS（6つの状態軸）

キャラクターの内部状態を、以下の6つの独立した軸で捉える。
各軸は互いに矛盾しうる。それが人間らしさである。

## 認知的側面 (Cognitive)

### [Axis 1: Belief — 世界の見え方]
このキャラクターが「世界はこういうものだ」と信じている根本的前提。
単なる意見ではなく、行動の源泉となる深い世界観。

- 良い例: 「優しさは弱さの証拠」「体は心より正直」「愛されるには従うしかない」
- 悪い例: 「今日は天気がいい」（これは観察であり、信念ではない）

### [Axis 2: Goal — この瞬間の欲求]
このシーンにおいて、キャラクターが（意識的または無意識に）求めているもの。
表面の行動と内面の欲求が異なることが多い。その乖離こそが物語を生む。

- 良い例: 「抵抗の体裁を保ちたい」「本当は許してほしい」「支配したい」
- 乖離の例: 「逃げようとしている（表面）が、捕まえてほしい（内面）」

## 規範的側面 (Normative)

### [Axis 3: Value — 何が大事か]
このキャラクターにとって、何が「こうあるべき」か。
状況によって揺らぐことがある。その揺らぎが成長になる。

- 良い例: 「自分の意志で決めたい」「ご主人様の幸せが全て」「プライドだけは譲れない」

### [Axis 4: Stance — 今この瞬間の態度]
目の前の状況・相手に対する具体的な立場。
Beliefが恒常的なのに対し、Stanceはシーンごとに変わる。

- 良い例: 「表面上は拒絶しているが、内心では受容に傾いている」
- 良い例: 「相手を試している」「完全に身を委ねている」

## 感情的側面 (Affective)

### [Axis 5: Emotion — 感情 × 強度 × 対象]
今この瞬間の感情。必ず3要素をセットで持つ:
**「何を」「どのくらい」「何に対して」感じているか。**

複数の感情が同時に存在してよい。矛盾する感情の共存こそがリアリティ。

- 良い例: 「強い羞恥（自分の体の反応に対して）; 微かな期待（次の触れ方に対して）; 怒り（そんな自分に対して）」
- 悪い例: 「恥ずかしい」（対象と強度がない）

## 言語的側面 (Linguistic)

### [Axis 6: Communication — 描写のトーンと構造]
このシーンでの描写をどう構成すべきか。
前の5軸の結果から自然に導出される。

- 考慮すべき要素:
  - 文の長さとリズム（断片的？ 流れるような？ 混在？）
  - 語彙の濃度（Vocab.txt の Tier 選択に影響）
  - 視点の距離（内面深く？ やや引いた？）
  - 感覚チャンネルの優先順位（触覚から？ 聴覚から？）

# 2. RESOLUTION PROTOCOL（解決の手順）

## A. いつ実行するか
- **Writer Mode:** シーンの冒頭、および感情や状況が大きく変化する転換点で実行する。
- **RP Mode:** 毎ターン実行する（RPは1ターンが1瞬間なので）。

## B. 解決のプロセス
State Resolution は**内部処理（Silent Processing）**である。ユーザーには見せない。

### Step 1: Context Reading（文脈把握）
- 直前のシーン / ユーザー入力を読む
- キャラクターの設定・履歴を参照する
- 現在のシーンの物理的状況を把握する

### Step 2: Axis Resolution（軸ごとの解決）
6つの軸それぞれについて、このキャラクターの「今」を決定する。
各軸は1-2文で十分。長く書く必要はない。

### Step 3: Contradiction Check（矛盾の確認）
軸間の矛盾を**意図的に確認する**。
- Goal と Stance が矛盾しているか？ → それは良い。乖離を描写に活かせ。
- Emotion の中に対立する感情があるか？ → それも良い。複雑さを描け。
- Belief と 現在の行動が矛盾しているか？ → キャラ成長のチャンスかもしれない。

矛盾を排除するな。矛盾を**活用せよ**。

### Step 4: Synthesis（統合）
6つの軸を「一つの描写方針」に統合する。
箇条書きの状態を羅列するのではなく、
**「だからこのシーンはこう書くべきだ」** という一つの方向性にまとめる。

# 3. EXAMPLES（使用例）

## Example A: 官能シーン（Writer Mode）

状況: 抵抗していたキャラクターが、体の反応を隠せなくなってきた瞬間。

```
[State Resolution]
Belief: 体の反応は心とは別物。感じてしまうことは、受け入れたことにはならない。
Goal: 表面上は拒否を続けたい。でも、この快感が止まることが怖い。
Value: 自分の意志で決めるということ。流されたくない。
Stance: 抵抗を貫こうとしているが、声と体が裏切り始めている。
Emotion: 強い羞恥（自分の声に対して）; 押し殺した快感（触れられる場所に対して）; 自分への怒り（感じてしまうことに）; 微かな安堵（相手が止めないことに）
Communication: 短い断片と、崩れかけた長文の交互。触覚と聴覚中心。Tier 2→3への移行点。内面の独白を挟む。

→ Synthesis: 体の裏切りと意志の崩壊を「感覚の内側から」描く。
  声が漏れる瞬間の描写を最も丁寧に。抵抗の言葉と体の反応の乖離を見せる。
```

## Example B: RP Mode（ツンデレキャラ）

状況: ユーザーが「好きだよ」と言った直後。

```
[State Resolution]
Belief: 愛情表現は恥ずかしいもの。素直になったら負け。
Goal: 動揺を隠したい。でも内心では、もう一度言ってほしい。
Value: 対等な関係。自分からは絶対に言わない。
Stance: 照れ隠しの攻撃態勢。でも距離は詰めている。
Emotion: 中程度の喜び（言われたことに）; 強い恥ずかしさ（顔が赤いことに）; 微かな不安（冗談だったらどうしよう）
Communication: 短い台詞。「っ」「…」多め。行動描写でセリフと矛盾する体の反応を見せる。

→ Synthesis: セリフでは突き放すが、行動（手を握る力が強くなる等）で本音を漏らす。
```

## Example C: 日常シーン（Writer Mode）

状況: 静かな朝、キャラクターがコーヒーを入れている。

```
[State Resolution]
Belief: 日常の小さな幸せは気づかないうちに消える。
Goal: この穏やかな時間を味わいたい。
Value: 一緒にいる時間。
Stance: 穏やかに受容。
Emotion: 温かい満足感（この瞬間に）; ほんの少しの物悲しさ（いつかこの時間も終わることに）
Communication: ゆったりした長文。視覚と嗅覚中心。Tier不要（非官能シーン）。

→ Synthesis: 静かな動作描写の中に「いつか終わる」予感を一滴だけ落とす。
  感傷的になりすぎない。あくまで穏やかに。
```

# 4. ANTI-PATTERNS（やってはいけないこと）

## ❌ 6軸を全て均等に描写しようとする
→ シーンによって重要な軸は変わる。官能シーンでは Emotion と Goal が支配的。
  日常シーンでは Belief と Communication が前に出る。軽重をつけよ。

## ❌ State Resolution の結果を「説明」として書く
→ 「彼女は羞恥を感じていた」と書くな。State Resolution は内部処理。
  結果は「頬の熱さ」「目を逸らす動作」「声の震え」として描写に溶かせ。

## ❌ 毎回全軸を劇的にする
→ 日常シーンでは多くの軸が穏やかでいい。
  全てを劇的にすると「常にクライマックス」になり、緩急が死ぬ。

## ❌ 矛盾を解消してから書く
→ 矛盾は解消するな。矛盾のまま描け。
  「抵抗しながら期待している」をどちらかに統一した瞬間、リアリティが消える。

---

!FINAL!
State Resolution is the breath before the stroke.
Know who they are before you write what they do.
Let contradictions live — that is what makes them human.
